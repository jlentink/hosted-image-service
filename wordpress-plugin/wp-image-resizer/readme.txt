Offloads image resizing and cropping to an external high-performance image-service.

== Description ==

WP Image Resizer replaces WordPress's built-in image processing (GD/Imagick) with an external Go-based image service powered by libvips.

**Features:**

* 4-8x faster image processing via libvips
* Output to JPEG, PNG, WebP, and AVIF
* Smart crop (attention-based, finds most interesting area)
* Focal point crop — click to set a focal point directly in the media library
* Bulk regenerate all thumbnails via Media › Regenerate Thumbnails or the media library bulk action
* JWT authentication with shared secret
* Compatible with all standard WordPress image sizes
* Live connection status check on the settings page

**Requirements:**

* A running instance of [image-service](https://github.com/jlentink/image-service)
* PHP 7.4+
* WordPress 5.8+

== Installation ==

1. Deploy the image-service (see the [deployment guide](https://github.com/jlentink/image-service/blob/main/docs/DEPLOYMENT.md))
2. Upload the `wp-image-resizer` folder to `/wp-content/plugins/`
3. Activate the plugin through the **Plugins** menu in WordPress
4. Go to **Settings › Image Resizer**
5. Enter your service URL and JWT shared secret (must match `auth.jwt_secret` in your image-service config)
6. Check **Enable** to activate image processing
7. Click **Save Changes** — a green "Service is reachable" message confirms the connection
8. Upload an image to test

== Configuration ==

Navigate to **Settings › Image Resizer** to configure the plugin.

**Service Connection**

* **Enable** — Check to activate the external image service. Image processing falls back to WordPress defaults when unchecked.
* **Service URL** — Full URL to your image-service instance (e.g. `https://images.example.com`). No trailing slash.
* **JWT Secret** — Shared secret used for authentication. Must exactly match `auth.jwt_secret` in your image-service TOML config.

**Image Quality**

| Setting | Default | Range | Notes |
|---|---|---|---|
| Default Format | JPEG | JPEG, PNG, WebP, AVIF | Output format for all resized images |
| JPEG Quality | 85 | 1–100 | Higher = better quality, larger file |
| WebP Quality | 80 | 1–100 | Higher = better quality, larger file |
| AVIF Quality | 60 | 1–100 | AVIF achieves good quality at lower values |
| PNG Compression | 6 | 0–9 | Higher = smaller file, slower to compress |

**Crop Modes (per image)**

Set per-image in the media library attachment panel:

* **Center** — Crops to the center of the image (default)
* **Smart** — Uses libvips attention-based detection to find the most interesting area
* **Focal Point** — Click on the image thumbnail to pin a focal point; the service crops around that coordinate

Changes take effect immediately when you click **Save & Regenerate**, or on the next thumbnail regeneration.

**Regenerating Thumbnails**

* **Bulk via admin page** — Go to **Media › Regenerate Thumbnails** and click "Regenerate All Thumbnails". A progress bar shows status.
* **Bulk via media library** — Select images in the media library, choose "Regenerate Thumbnails (Image Service)" from the Bulk Actions dropdown.

== Frequently Asked Questions ==

= Images are not being processed by the service. Why? =

Check that the plugin is both activated *and* enabled. Activation installs the plugin; enabling it (the **Enable** checkbox in Settings › Image Resizer) is a separate step. You also need a valid Service URL and JWT Secret saved before enabling.

= The settings page shows "Service is not reachable." =

The plugin performs a GET to `/health` on your service URL. Common causes: the service is not running, the URL has a trailing slash or typo, or a firewall/network rule blocks outbound requests from your WordPress server to the image-service host.

= Do I need to regenerate existing thumbnails after installing? =

Yes. The plugin only processes images at the time they are resized. Existing thumbnails in the media library were generated by GD/Imagick and will not be replaced until you regenerate them via **Media › Regenerate Thumbnails**.

= Can I disable SSL verification for a local or self-signed setup? =

Add this to your theme's `functions.php` or a custom plugin:

`add_filter( 'wpir_sslverify', '__return_false' );`

Do not use this in production.

= What happens if the image service is down? =

If the service returns an error or is unreachable, the individual thumbnail generation fails. WordPress logs a warning but does not prevent the upload from completing. The original full-size image is always preserved.

= Is the JWT secret stored securely? =

The secret is stored in the WordPress options table (same as any other plugin credential). Use a long, random string and ensure your database access is appropriately restricted.
